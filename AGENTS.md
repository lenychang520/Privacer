# Privacer — Agent Guide

## Project structure

```
core/              Rust library (edition 2024) — detection engine
  src/lib.rs        re-exports: config, detector, patterns, entropy, preprocess, whitelist
  src/detector.rs   PrivacyDetector class, filter_text/scan_text convenience fns, 51 tests
  src/patterns.rs   27 built-in regex rules with fancy-regex lookaround
  src/config.rs     PrivacerConfig (YAML-deserializable), EntropyMode, CustomRule
  src/preprocess.rs NFKC → iterative URL decode → HTML unescape → zero-width strip
  src/whitelist.rs  0.0.0.0, 255.255.255.255, localhost, example.com etc.
  src/entropy.rs    Shannon entropy sliding window
wasm/              Rust crate → wasm-bindgen → cdylib
  src/lib.rs        exports: filter(text, bool) -> PrivacerResult, scan(text, bool) -> usize
  pre-built WASM goes into vscode-extension/wasm/
vscode-extension/  TypeScript VS Code extension (CommonJS, tsc compiles src/ → .)
  src/extension.ts  monkey-patches http.ClientRequest#write/end for LLM paths
  wasm/privacer_wasm_bg.wasm    ~1.6MB pre-built binary (committed)
  wasm/privacer_wasm.js         JS glue (committed, generated by wasm-pack)
```

## opencode plugin setup

Privacer ships an opencode plugin that redacts sensitive data from all LLM-bound messages across every project.

```sh
# Quick install (global — works in any project)
bash scripts/setup-opencode-plugin.sh

# Restart opencode, then verify:
tail -f .privacer/logs/opencode-*.log
# Expected: Plugin initializing → WASM loaded → Plugin ready
```

The plugin is installed to `~/.config/opencode/plugins/privacer.js`.
Source lives at `scripts/privacer-plugin.js` — the WASM binary at `vscode-extension/wasm/`.

## Build & test

```sh
# All commands from repo root (or respective subdirectory)

# Core tests (51 tests)
cargo test -p privacer-core

# Single test
cargo test -p privacer-core test_name

# Build core library
cargo build -p privacer-core

# Build WASM (requires wasm-pack)
cd wasm && wasm-pack build --target nodejs --out-dir ../vscode-extension/wasm --no-opt

# Build VS Code extension
cd vscode-extension && npx tsc && npx @vscode/vsce package --allow-missing-repository
```

No lint, typecheck, or CI scripts exist. `cargo test` is the only verification.

## WASM API (consumed by JS/TS/Python)

```ts
filter(text: string, enable_entropy: boolean): { text: string, replacements: number }
scan(text: string, enable_entropy: boolean): number  // match count
```

Pre-built in `vscode-extension/wasm/` — do not regen unless core changes.

## DetectionConfig (convenience API) vs PrivacerConfig (full config)

- `DetectionConfig` — simple struct: entropy on/off, threshold, min_length. Used by `filter_text()` / `scan_text()` and WASM layer.
- `PrivacerConfig` — full YAML config: rule toggles, custom rules, placeholder overrides, whitelist extensions, preprocess toggles, entropy mode (Auto/Review). Used by `PrivacyDetector::new(config)`.

### What is NOT detected (known limitations)
- IPv4 with bracket obfuscation: `192[.]168[.]1[.]1`
- IPv4 with spaces: `192. 168. 1. 1`
- Email with "at" bypass: `user at company dot com`

### Whitelisted (never filtered)
- IPs: `0.0.0.0`, `255.255.255.255`, RFC 1918 (`10.x.x.x`, `172.16-31.x.x`, `192.168.x.x`)
- Domains: `localhost`, `example.com`, `example.org`, `test.com`, `test.local`
- Hostnames: `localhost`

## Key conventions

- `fancy-regex` for built-in regex patterns (lookaround support). `regex` crate for utility patterns and custom rules.
- Rust edition 2024 — bare `impl Trait` in return position, `unsafe_op_in_unsafe_fn` required.
- All core detection goes through `filter_text()` / `scan_text()` convenience fns OR `PrivacyDetector` class.
- Overlap dedup: sort by start → priority → end descending; full containment always wins.
- Credit card Luhn failure → `confidence: Low` (scan still returns, filter skips).
- IPv6 regex skipped on text >5000 chars (ReDoS guard).
- Max input: 100KB.
- Config file loading (`PrivacerConfig::load_from_path()`) uses `serde_yaml`. Not used by WASM path.

## Logging

Privacer writes logs to `.privacer/logs/` (relative to the project root).

| Field | What it means |
|-------|---------------|
| `[INFO] Plugin initializing, project: xxx` | Plugin was loaded by the host platform |
| `[INFO] WASM loaded from /path` | WASM engine found and initialized |
| `[WARN] WASM not found` | No WASM binary found — filtering is disabled |
| `[INFO] Redacted N item(s) from message` | N sensitive items were detected and replaced |
| `[ERROR] Filter failed: ...` | WASM filter threw an exception (should not happen) |

Log files rotate daily: `opencode-YYYY-MM-DD.log`.

### How to verify the plugin is active
1. Check for `.privacer/logs/` directory
2. Look for `Plugin initializing` + `WASM loaded` lines
3. Send a test IP (e.g. `192.168.1.1`) — expect `Redacted 1 item(s)` line and `[IP]` in the AI's view
